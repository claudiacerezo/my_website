The Bechdel Test

https://fivethirtyeight.com/features/the-dollar-and-cents-case-against-hollywoods-exclusion-of-women/

The Bechdel test is a way to assess how women are depicted in Hollywood movies. In order for a movie to pass the test:

1.         It has to have at least two [named] women in it

2.         Who talk to each other

3.         About something besides a man

There is a nice article and analysis you can find here https://fivethirtyeight.com/features/the-dollar-and-cents-case-against-hollywoods-exclusion-of-women/ We have a sample of 1394 movies and we want to fit a model to predict whether a film passes the test or not.

step_novel()

Adds a catch-all level to a factor for any new values not encountered in model training, which lets R intelligently predict new levels in the test set.

step_zv()

Intelligently handles zero variance variables (variables that contain only a single value)

movie_rec <- recipe(test ~ ., data = bechdel) %>%

1.         How many models have you specified? We have specified 5 models.

2.         Whats the difference between a model specification and a workflow?

A model specification gives a detailed description of a model. It includes information about the model type, the variables used as inputs, the variable to be predicted (target or response variable), and any specific assumptions or constraints associated with the model. The model specification defines the mathematical or algorithmic representation of the model and serves as a blueprint for building or fitting the model to the data.

A workflow is the steps involved in a project. It includes the entire process from data preparation and preprocessing to model building, evaluation, and deployment. A workflow includes tasks such as data cleaning, feature engineering, model selection, hyperparameter tuning, model validation, and result interpretation. It outlines the overall structure and order of operations for a given project.

3.         Do you need to add a formula (e.g., test ~ .) if you have a recipe?

If I have a recipe that defines the necessary data transformations and feature engineering steps, I dont need to include a formula like test ~ .. The formula syntax is used to specify the relationship between variables in a dataset. However, if I have a recipe that already encodes the necessary transformations and feature engineering steps, the formula might be unnecessary as the transformed variables will already be available for model fitting.
